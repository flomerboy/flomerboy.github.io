Ryan uses tools

I'm using this space to document all the tools I do and don't use. Not all of these definitions make a ton of sense, or are 100% clear, but I think it's good to keep track and be critical of my process.

Design:
  • Illustrator: For making icons, illustrations, compositions
  • Sketch: For making UIs and literally nothing else
  • Form: For prototyping motion of elements in a UI
  • InDesign: For multi-page print stuff or posters
  • Photoshop: For editing large amounts of pixels to deceive humans
Web Dev:
  • Sublime Text 2: Text Editor
  • Ionic: Lets me make native apps with web dev stack. Built on Angular
  • HTML/CSS/Javascript: static webpages
  • Angular.js: Front end data presentation for dynamic web apps, extends HTML
  • Node.js: Server-side data moving
  • Homebrew: Missing package manager for OSX
  • Bower: package management for front-end
  • MEAN.io: full-stack for modern web apps. (mongoDB>Express.js>Angular.js>Node.js)
  • screenGrabber, a free, automated screen shot app :)
